# エモ→ロジ・メソッド 統合テストケース

## 1. ドキュメント構造の整合性テスト

### 1.1 基本構造の検証
- [ ] メソッドの目的が明確に定義されているか
- [ ] 適用範囲が明示されているか
- [ ] 前提条件が明確に記載されているか
- [ ] 各ステップが論理的に整理されているか

### 1.2 ファイル間の一貫性
- [ ] 用語の使用が統一されているか
- [ ] ステップの順序が一致しているか
- [ ] 概念の説明に矛盾がないか
- [ ] フォーマットが統一されているか

## 2. コンテンツの完全性テスト

### 2.1 創作版の要素網羅
- [ ] 感情優先のアプローチが保持されているか
- [ ] ストーリー創作特有の要素が含まれているか
- [ ] インパクトと整合性のバランスが説明されているか
- [ ] 具体的な創作例や指針が含まれているか

### 2.2 汎用版の要素網羅
- [ ] ビジネス応用の視点が含まれているか
- [ ] 評価基準が明確に定義されているか
- [ ] 品質管理の方法が説明されているか
- [ ] 継続的改善の仕組みが含まれているか

## 3. 実用性テスト

### 3.1 手順の実行可能性
- [ ] 各ステップが具体的で実行可能か
- [ ] 必要なインプット・アウトプットが明確か
- [ ] 評価基準が測定可能か
- [ ] フィードバックの仕組みが組み込まれているか

### 3.2 汎用性の確保
- [ ] 様々な分野への応用が可能か
- [ ] カスタマイズの方法が説明されているか
- [ ] 異なる規模のプロジェクトに対応できるか
- [ ] 他の手法との統合方法が示されているか

## 4. 理解容易性テスト

### 4.1 文書の可読性
- [ ] 説明が明確で分かりやすいか
- [ ] 専門用語の説明が適切か
- [ ] 構造が論理的に整理されているか
- [ ] 視覚的な要素（図表など）が効果的に使用されているか

### 4.2 実践的な例示
- [ ] 具体例が十分に提供されているか
- [ ] 応用例が多様な分野をカバーしているか
- [ ] 失敗例や注意点が含まれているか
- [ ] ベストプラクティスが示されているか

## 5. 両版の統合性テスト

### 5.1 コア概念の一貫性
- [ ] エモーショナル・ブレインストーミングの本質が保持されているか
- [ ] コアアイデアの設計プロセスが統一されているか
- [ ] ロジカル・アウトラインの考え方が一致しているか
- [ ] 感情と論理のバランス取りの方法が統一されているか

### 5.2 相互補完性
- [ ] 創作版と汎用版が互いに補完し合っているか
- [ ] 異なる視点からの知見が統合されているか
- [ ] 応用範囲が適切に拡張されているか
- [ ] 両者の強みが活かされているか
